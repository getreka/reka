## Changes Deployed (2026-04-09/10)

### Security
- Auth deny-by-default: no keys → 401 (was: open access)
- /metrics removed from skip-auth paths
- Rate limiter moved before auth middleware (blocks brute-force)
- API keys migrated from legacy UUID to `rk_{project}_{hex}` format
- @getreka/mcp published with REKA_API_KEY support

### Performance
- Ollama: qwen3.5:35b → qwen3.5:9b (14 → 95 tok/s, 75% error rate → 0%)
- OLLAMA_KEEP_ALIVE=-1 (no model unload between requests)
- end_session: 46s blocking → <1s (consolidation moved to async worker)
- end_session MCP timeout: 30s → 120s

### Memory Pipeline
- Sensory ingestion added for start_session, end_session, consolidation events
- remember/batch_remember salience: 0.4 → 0.7 (now reaches working memory + consolidation)
- Consolidation moved from endSession sync path to BullMQ session-lifecycle worker
- Short-circuit: skip LLM consolidation for short sessions (< 5 events) with manual remember
- Stale session cleanup now triggers consolidation via session:ending event
- Redis eviction policy: allkeys-lru → noeviction

### Cleanup
- Removed dead feedback/search POST (404 spam in logs)
- npm published @getreka/mcp@0.2.0 with all fixes
- Auth tests updated for new behavior (9/9 pass)
